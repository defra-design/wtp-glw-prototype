Hypotheses

Journey
For agency officers we believe that:
Overarching

  • Using more universal terms for the retrieval journey will not be detrimental to understanding the function of the service

    • Validated
    Tests
    • replace "retrieval" with "find"

  • Creating separate journeys to find the correct documents will not impare the user's experience

    • Validated
    Tests
    • create a new, specific card on the start page to find a specific document type.
Document archiving

  1. The ability to view the whole page preview will provide additional confidence whilst locating official numbers to key-in

    2. Validated
    Tests
    • view full preview feature and test with cancellation document

  2. Providing meaningful system feedback when issues are enountered will help solve problems and provide a better end-to-end experience

    3. More research needed
    Tests

      Scenario

    • It's your turn on the rota and a large number of documents have been delivered to process. There has been reports of network problems, it's going to be one of those days...
    • no scanners found Validated
    • scanner busy Validated
    • problem scanning document More research needed API messages
    • official number error Validated Accessibility testing needed
    • 500 - Internal server error Validated Remove clear cache / browser guidance
Find documents

  1. Separate journeys for finding a specific docuemnt will simplify the retrieval interface and yield more specific results

    Validated
    2. Tests
    • find first document type - reg#, results, simplify label
    • find second document type - reg#, reg date, results, simplify label

  2. Editing existing records can be sign-posted via the respective search results page

    Validated
    3. Tests
    • results page with an edit feature enabling information to be updated and saved
    • for other document types, create a new version of the check your answers page to include registration number, date view preview

  3. Surfacing additional information on a cover sheet will be easier to navigate with assistive tech than embeded metadata within a PDF

    More research needed
    4. Tests
    With screen reader, test the end-to-end journey but focus testing with PDF varients:
    1. A flat scanned document
    2. Tagged cover sheet content with scanned images
    3. Scanned images with embedded meta data
